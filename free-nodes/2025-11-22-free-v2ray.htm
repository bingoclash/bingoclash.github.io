一、Quantumult X 是什么？它的定位远超代理工具

Quantumult X（以下简称QX）不仅仅是一个普通的代理客户端，它是一套网络管理解决方案。其核心定位是——为用户提供完整、透明、可控的网络请求处理逻辑。

在安卓版本中，QX保留了大部分iOS核心功能，并加入了一些平台特性，让安卓用户也能享受到高级代理技术带来的便利与自由。

核心能力概览：

  • 支持多协议代理：包括 HTTP、HTTPS、SOCKS5、Shadowsocks、Vmess 等。

  • 灵活的规则配置系统：支持根据域名、IP、关键词等设定行为逻辑。

  • 自定义 DNS 服务：提高解析速度，绕过DNS污染。

  • 分流与重定向支持：可将不同应用或网站的流量分配给不同的节点或策略。

  • 实时流量监控和日志记录：网络行为透明可见，方便排错与优化。

  • 自动更新订阅与规则集：支持外部订阅链接，轻松实现节点批量管理。

在功能层级上，QX几乎达到了商用级别的网络管理水平，却以极简的方式呈现给终端用户，可谓“简洁中见锋芒”。

三、Quantumult X 配置实战：从零开始掌握核心能力

真正让Quantumult X发挥价值的，是它强大的配置系统。以下将从基础代理设定开始，一步步讲解如何打造属于你的QX网络规则体系。

1. 添加代理服务器节点（节点来源）

要实现科学上网，首先需要代理节点。节点来源一般有三种：

  • 购买付费节点订阅（推荐）：如Clash、V2ray、Trojan等服务商提供的订阅链接，稳定、安全。

  • 使用免费分享节点：不稳定，安全性低，适用于测试和临时访问。

  • 自建节点：成本高但可控，适合高阶用户。

导入方式：

  • 打开Quantumult X主界面，点击右上角“+”，选择“通过订阅链接导入”；

  • 粘贴你的订阅链接；

  • 导入后会自动更新节点列表。

2. 设置DNS服务器

DNS 是网络访问中的第一步，决定了你能否顺利访问目标网站。

推荐配置：

  • Cloudflare DNS：1.1.1.1 / 1.0.0.1

  • Google DNS：8.8.8.8 / 8.8.4.4

在设置中手动输入DNS地址，建议开启 DNS-over-HTTPS 或 DNS-over-TLS，防止DNS污染。

3. 自定义规则分流

这是Quantumult X最有价值的部分。你可以将如下流量自动分流：

  • 国内网站 → 直连

  • 国际网站（如Twitter、YouTube）→ 走代理

  • 广告域名 → 拦截

  • 游戏类请求 → 分配特定节点（如低延迟日本节点）

示例规则：

ini
[filter]
IP-CIDR,192.168.0.0/16,DIRECT
DOMAIN-SUFFIX,twitter.com,Proxy
DOMAIN-KEYWORD,google,Proxy
GEOIP,CN,DIRECT
FINAL,Proxy

规则执行顺序从上到下，最后一条是“兜底规则”，未命中其他规则的流量将默认走 Proxy。

4. 快速启用和切换代理

在主界面中可快捷开启/关闭代理服务，还可在不同代理策略之间切换。

建议模式：

  • 工作时：规则分流

  • 下载大文件或看视频时：全局代理

  • 网络调试时：直连或记录日志模式

FAQ

Nekoray 节点订阅如何实现自动更新？
用户可添加订阅 URL，Nekoray 会定期自动下载并更新节点列表。保持节点最新可减少手动维护，提高节点可用性和网络连接稳定性，确保科学上网体验顺畅。
Mellow 与 Clash 的核心区别是什么？
Mellow 采用基于透明代理的架构，适合路由或系统级流量重定向，而 Clash 更偏向规则式应用代理。Mellow 的灵活性更高，但配置复杂；Clash 则兼顾易用性与功能扩展性。
SSR 的 UDP 转发功能如何启用？
SSR 支持 UDP 转发，可通过客户端设置开启。启用后，游戏和语音等 UDP 流量可通过节点代理，降低丢包率和延迟，提高实时网络应用体验。
WinXray 如何导入订阅？
在 WinXray 界面中点击“导入订阅”选项，将机场提供的订阅链接粘贴进去。点击更新后，WinXray 会自动下载节点配置并生成列表，用户可手动选择或自动切换延迟最低的节点使用。
Hysteria2 的多路复用在短连接场景下有什么优势？
多路复用允许多个短连接共享同一通道，减少握手延迟和资源消耗。适合高频短连接应用，如游戏和即时通讯，能提高吞吐量和连接稳定性，优化网络体验。
SSR 的端口复用有什么风险？
SSR 支持多端口配置以复用服务，但若端口过多或设置冲突，可能被扫描工具识别或误判为异常服务。建议保持配置简洁，并合理分配端口范围以减少暴露风险。
V2Ray 的 WebSocket 传输与 TCP 有何区别？
WebSocket 通过 HTTP 封装 TCP 流量，适合伪装成正常网页请求，增加隐蔽性。相比原生 TCP，可更好地绕过封锁和 DPI 检测，但略增加协议开销。
WinXray 的多节点同时使用会影响性能吗？
同时使用多个节点会占用更多系统资源和带宽，可能对性能有一定影响。通过策略组和分流规则合理安排流量，可以降低资源占用，同时保持网络稳定和速度。
如何在 Nginx 上为 Vmess/WS 配置反向代理与 TLS？
在 Nginx 中配置 HTTPS 站点并将特定路径（如 /ray）反向代理到 V2Ray 的 WebSocket 端口，确保 proxy_set_header 设置正确（Host、X-Real-IP 等）。同时使用 certbot 获取并自动续期 TLS 证书以维护安全。
如何使用 Sing-box 配合 Clash Rules 实现精细分流？
将 Clash 的 rules 段提取出来转换为 Sing-box 的 routing 格式，或直接引用 rule-providers 源。Sing-box 的路由引擎支持更复杂的条件匹配，可结合 GeoIP 与 keyword 实现精细分流策略。
